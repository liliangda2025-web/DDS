## 1. 通用约定

### 1.1 基础路径

```http
/api/knowledge
```

其中 `/api` 由 `application.yml` 中的 `server.servlet.context-path` 统一配置，Controller 映射只声明
`/knowledge` 及其子路径，不重复书写 `/api`。

### 1.2 通用响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 1.3 HTTP 状态与业务码

1. 成功响应使用 HTTP 200，响应体业务码也使用 `200`。
2. 参数错误使用 HTTP 400 和业务码 `40000`。
3. 通用业务错误使用 HTTP 400 和业务码 `40001`。
4. 未认证、无权限、资源不存在、方法不支持分别使用 HTTP 401、403、404、405。
5. 未知系统异常使用 HTTP 500 和业务码 `50000`，不向前端返回内部异常信息。

### 1.4 分页响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 35,
    "pageNum": 1,
    "pageSize": 10,
    "pages": 4,
    "records": []
  }
}
```

### 1.5 ID 约定

知识条目 ID 和文件 ID 建议前端按字符串处理，避免 JavaScript 大整数精度丢失。

| 字段    | 含义                      |
| ------- | ------------------------- |
| entryId | 知识条目 ID，对应知识条目 |
| fileId  | 文件 ID，对应具体文件记录 |

示例：

```json
{
  "entryId": "2100000000000000001",
  "fileId": "2200000000000000001"
}
```

### 1.6 知识条目类型枚举

| 值           | 含义         |
| ------------ | ------------ |
| DRAWING      | 图纸库       |
| PROGRAM_RULE | 程序生效准则 |
| LAW          | 法律法规     |
| CASE         | 历史案例     |

### 1.7 信息完善状态枚举

| 值   | 含义   |
| ---- | ------ |
| 0    | 待补充 |
| 1    | 已完善 |

### 1.8 文件上传状态枚举

| 值      | 含义     |
| ------- | -------- |
| success | 上传成功 |
| failed  | 上传失败 |

# 3. 知识条目管理接口

## 3.1 分页查询知识条目

### 接口说明

用于知识条目管理页面的表格列表查询。

该接口返回知识条目信息及其当前有效文件信息。

### 请求方式

```http
POST /api/knowledge/entries/page
```

### Content-Type

```http
application/json
```

### 请求参数

| 字段           | 类型   | 是否必填 | 说明                             |
| -------------- | ------ | -------- | -------------------------------- |
| entryType      | string | 否       | 知识条目类型                     |
| fileName       | string | 否       | 文件名，模糊查询                 |
| entryCode      | string | 否       | 条目编码 / 图纸编码              |
| title          | string | 否       | 标题，模糊查询                   |
| professionCode | string | 否       | 专业代码                         |
| infoStatus     | number | 否       | 信息完善状态：0 待补充，1 已完善 |
| startDate      | string | 否       | 发版开始日期，格式 yyyy-MM-dd    |

| pageSize       | number | 否       | 每页条数，默认 10                |

## 3.5 删除知识条目

### 接口说明

用于删除知识条目。

删除采用逻辑删除。删除知识条目后，其关联文件也不再作为有效文件展示。

删除时，后端必须在同一事务中将知识条目的 `deleteMarker` 更新为当前 `entryId`、将 `status`
更新为 0，并将关联文件状态更新为 0。删除后允许重新创建相同类型、编码和版本的条目。

### 请求方式

```http
DELETE /api/knowledge/entries/{entryId}
```

### 路径参数

| 字段    | 类型   | 是否必填 | 说明        |
| ------- | ------ | -------- | ----------- |
| entryId | string | 是       | 知识条目 ID |

### 响应示例

```json
{
  "code": 200,
  "message": "删除成功",
  "data": true
}
```

# 4. 文件管理接口

## 4.1 上传或替换当前文件

### 接口说明

用于给某个知识条目上传当前文件。

如果该知识条目已经存在当前有效文件，则执行替换逻辑：旧文件变为非当前文件，新文件成为当前有效文件。

### 请求方式

```http
POST /api/knowledge/entries/{entryId}/files/current
```

### Content-Type

```http
multipart/form-data
```

### 路径参数

| 字段    | 类型   | 是否必填 | 说明        |
| ------- | ------ | -------- | ----------- |
| entryId | string | 是       | 知识条目 ID |

### 表单参数

| 字段 | 类型 | 是否必填 | 说明       |
| ---- | ---- | -------- | ---------- |
| file | file | 是       | 上传的文件 |

### 响应示例

```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "fileId": "2200000000000000001",
    "entryId": "2100000000000000001",
    "originalFileName": "总部办公楼暖通空调设计图1.pdf",
    "fileType": "pdf",
    "fileSize": 2048000,
    "fileUrl": "/uploads/knowledge/2100000000000000001.pdf",
    "uploadStatus": "success",
    "isCurrent": 1,
    "uploadedAt": "2026-06-30 10:00:00"
  }
}
```

### 业务规则

1. `entryId` 必须存在。
2. 文件不能为空。
3. 当前阶段主要支持 PDF 文件。
4. 上传成功后，新文件成为当前有效文件。
5. 如果原来存在当前有效文件，原文件应变为非当前文件。
6. 上传失败时返回失败原因。

# 5. 批量导入文件接口

## 5.1 批量导入文件

### 接口说明

用于“批量导入文件”功能。

用户只选择知识条目类型并上传多个文件。系统为每个上传成功的文件生成一条待补充知识条目和一条知识文件记录。

### 请求方式

```http
POST /api/knowledge/files/batch-upload
```

### Content-Type

```http
multipart/form-data
```

### 表单参数

| 字段      | 类型   | 是否必填 | 说明         |
| --------- | ------ | -------- | ------------ |
| entryType | string | 是       | 知识条目类型 |
| files     | file[] | 是       | 多个文件     |

### 请求示例

```text
entryType = DRAWING
files = [a.pdf, b.pdf, c.pdf]
```

### 响应字段

| 字段         | 类型   | 说明             |
| ------------ | ------ | ---------------- |
| totalCount   | number | 文件总数         |
| successCount | number | 成功数量         |
| failedCount  | number | 失败数量         |
| fileList     | array  | 每个文件处理结果 |

### fileList 字段说明

| 字段         | 类型   | 说明              |
| ------------ | ------ | ----------------- |
| index        | number | 序号              |
| entryId      | string | 生成的知识条目 ID |
| fileId       | string | 生成的文件 ID     |
| fileName     | string | 文件名            |
| fileType     | string | 文件类型          |
| fileSize     | number | 文件大小          |
| fileUrl      | string | 文件访问地址      |
| uploadStatus | string | 上传状态          |
| infoStatus   | number | 知识条目信息状态  |
| uploadedAt   | string | 上传时间          |
| errorMsg     | string | 失败原因          |

### 响应示例

```json
{
  "code": 200,
  "message": "批量上传完成",
```

# 6. 导出接口

## 6.1 导出知识条目列表

### 接口说明

用于知识条目管理页面点击“导出”。

导出的 Excel 用于用户手工补充知识条目信息，后续可通过“批量导入知识条目信息”接口回填系统。

### 请求方式

```http
POST /api/knowledge/entries/export
```

### Content-Type

```http
application/json
```

### 请求参数

与“分页查询知识条目”查询条件一致，但不需要分页参数。

| 字段           | 类型   | 是否必填 | 说明                |
| -------------- | ------ | -------- | ------------------- |
| entryType      | string | 否       | 知识条目类型        |
| fileName       | string | 否       | 文件名              |
| entryCode      | string | 否       | 条目编码 / 图纸编码 |
| title          | string | 否       | 标题                |
| professionCode | string | 否       | 专业代码            |
| infoStatus     | number | 否       | 信息完善状态        |
| startDate      | string | 否       | 发版开始日期        |
| endDate        | string | 否       | 发版结束日期        |

`startDate` 和 `endDate` 均过滤 `releaseDate`，采用闭区间。

### 请求示例

```json
{
  "entryType": "DRAWING",
  "fileName": "pdf",
  "infoStatus": 0,
  "startDate": "2026-06-01",
  "endDate": "2026-06-30"
}
```

### 响应类型

```http
application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
```

### 响应头示例

```http
Content-Disposition: attachment; filename="knowledge_entries.xlsx"
```

### Excel 导出字段

| Excel 列            | 说明                           |
| ------------------- | ------------------------------ |
| knowledge_entry_id  | 知识条目 ID，用于回填定位      |
| knowledge_file_id   | 文件 ID，用于回填校验          |
| 文件名称            | 原始文件名                     |
| 文件访问地址        | 文件预览访问地址               |
| 发版日期            | 用户填写或修改                 |
| 版本                | 用户填写或修改                 |
| 文件编号 / 图纸编号 | 用户填写或修改，对应 entryCode |
| 标题                | 用户填写或修改，对应 title     |
| 关键词              | 用户填写或修改                 |
| 所属项目            | 用户填写或修改                 |
| 系统来源            | 用户填写或修改                 |
| 专业代码            | 用户填写或修改                 |
| 编写人              | 用户填写或修改                 |
| 密级                | 用户填写或修改                 |
| 知识条目类型        | 用户填写或修改                 |
